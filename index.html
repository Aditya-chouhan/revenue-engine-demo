The CRM system behind the data

Design: crm-system-design.md. Enforcement: src/crm/lifecycle.py and src/crm/routing.py — this is executable code, not just a design doc.

Lead routing is a weighted round-robin with a signal-score fast lane: leads scoring ≥8.0 skip the queue and route to whoever has the lowest open-SQL load. This isn't just asserted — tests/test_analytics.py checks it directly: rep-level lead counts land within 1.7% of an even three-way split across all 2,080 leads (683 / 692 / 705).

Segmentation that changes real behavior

Accounts score 0–100 on category + firmographic + signal fit (src/segmentation/icp.py), bucketing into Beachhead / Core ICP / Adjacent / Poor Fit. Beachhead accounts convert at 4.3× Core ICP's Lead→Won rate (1.62% vs. 0.38%, on 1,109 vs. 799 leads) — the segmentation actually moves the funnel numbers, it isn't just a label on a report.

Multi-touch attribution — the models disagree, on purpose

Full file: src/analytics/attribution.py — first-touch, last-touch, linear, and U-shaped (40/20/40) over ~8,400 timestamped multi-touch activities.

First-touch and linear both credit Outbound LinkedIn as the top channel ($14,050 / $11,498 attributed MRR), but last-touch credits Signal-scored inbound instead ($9,950) — which channel "wins" depends on which attribution model a RevOps team chooses to trust, a real strategy decision this demo makes visible instead of picking one model and hiding the disagreement.

An AI copilot that never fabricates

The Einstein Copilot tab answers four canonical RevOps questions ("why is pipeline down," "which stage is underperforming," "where should sales focus," "which segment converts best") by reading the exact JSON payload above — a real Claude API call if a key is supplied in the sidebar (never persisted), or an honestly-labeled deterministic analyst engine if not. Same output schema either way; the "why is pipeline down" handler explicitly checks whether pipeline is actually declining before answering, so a low-volume partial month can't fool it into reporting a false decline. Full file: src/copilot/revops_copilot.py.

What's not built (named, not hidden)

  • Category-specialized routing is designed in crm-system-design.md but deliberately not activated — not enough per-rep, per-category sample size yet to trust the win-rate difference isn't noise.
  • CAC and retention aren't modeled — no spend or renewal-event data exists in this dataset to compute them honestly.
  • The Kanban board is click-to-view, not drag-to-advance — dragging a card would need a custom Streamlit component wired to lifecycle.advance().
  • The CRM's header search bar is visual chrome, not a live search — the real search/filter surfaces are the List view filters on each tab.
